responsive redesign
- lanzhou ramen site


redesigning a website to make it more visually engaging
and address usability problems

context

One of my favorite restaurants back home is Lanzhou Ramen <3.
But unfortunately, unlike their noodles, the design of their site is not the greatest. So I decided to use html + css to give their site a slight make-over! The new site is also meant to be responsive, meaning that it should adjust its formatting to fit various screens (tested on web, tablet, and phone screen sizes).


problem

Immediately, looking at the original Lanzhou Ramen site, we can see some visible structural and usability flaws.


click image to see the original site!


I then examined the website using the WebAIM WAVE tool to find the accessibility issues it may have. Upon seeing the findings, a few issues that I was in agreement with were the low-contrast visuals as well as some redundancy issues the tool pointed out.

Overall, I found numerous usability, learnability, and memorability issues with the site, namely:


• Clicking on “home” refreshes page instead of just either 1) doing nothing or 2) moving the page back to the top where the main header is located.

• Text blends in with the background since the image is so saturated. Low contrast.

• The site is cluttered with information with no hierarchy; a lot of negative space that causes the page to look long while serving no specific purpose.

• The “About” section located at the bottom of the page when it should be introducing the restaurant closer to the top.

• In general, the structure is confusing and has no specific order or hierarchy which make it not memorable for the user.

• Footer has redundancy issues. (redundant elements include: repeated menu option, delivery options, and location of what was already shown above in the page)

• Previously, it was stated that the restaurant has two locations but only lists one in the footer. similarly, the restaurant has two locations but only 1 phone number.

steps

thus, in attempt to fix some of those issues, I began designing a new site with cleaner organization (more hierarchy), more engaging visual elements, and fewer usability issues.

step one:
lo-fi prototypes


I first began the design process by creating low-fidelity prototypes that aims to eliminate the structural and usability issues that existed in the original site.

updated format:

• clicking on home no longer refreshes the page but rather moves the page back to the main section.

• the header image becomes the background image and has increased transparency to increase contrast and make the main section title stand out.

• the middle section is now adjusted to be presented by a scrolling style with clear sections with subheadings to show hierarchy and better categorize by purpose. This scrolling organization also gets rid of negative space.

• “About” section now moved up to the top to introduce users to the restaurant immediately.

• moved location out of the footer since it is an important section and adjusted to accommodate multiple locations. also included phone numbers for both locations with label of which number is for which location.

 

step two:
creating a visual design guide


with the lo-fi prototypes to guide the structure of the redesigned site, I then created a design guide for the visual elements in the site such as colors, fonts, decor styles, etc.

Important Finding!

I found that the visual design guide was extremely useful to keep the site clean and cohesive as well as visually engaging without over-incorporating color palettes and design elements.

reflection

This redesigned site turned out really well! I thought I was able to stick to the visual design guide fairly consistently and used hierarchy to improve the overall aesthetic of the website. I found it fun and helpful to use Figma so that you have a foundation to build your website upon. However, the iPhone Figma file ended up being long which also meant that the mobile version of the site was probably a bit on the long side and could be a little difficult to read. So in the future, I will try to make the Figma file (and in extension, the action site) shorter, more readable, and more adjusted for mobile users.
